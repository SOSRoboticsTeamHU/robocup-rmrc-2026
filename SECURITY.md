# Security Policy

## Supported Versions

We support the current development branch for security updates.

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not** open a public issue.

Instead, contact the maintainers privately (e.g. via the repository owner's GitHub profile or team contact). Include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

We will respond as soon as possible and work with you on a fix and disclosure.
